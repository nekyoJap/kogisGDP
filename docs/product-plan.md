# kogisGDP プロダクト計画書

## ミッション

**「みんなが競輪上手くなって欲しい」**

競輪の予想力は、経験者の頭の中にしかない暗黙知に大きく依存している。
本プロダクトは、予想のプロセスそのものを可視化・記録・共有することで、
誰でも「考え方」を学べるプラットフォームを目指す。

---

## 解決したい課題

1. **予想の思考プロセスがブラックボックス**
   - 上手い人がなぜその予想に至ったのか、過程が見えない
   - 結果（的中/不的中）だけが共有され、学びにつながらない

2. **選手評価の基準が属人的**
   - 評価の軸（スピード、スタミナ、競輪IQなど）が人によってバラバラ
   - 体系的に選手を評価する方法が共有されていない

3. **初心者が上達するための道筋がない**
   - 予想の組み立て方を段階的に学べる教材がない
   - 「何を見るべきか」すら分からない状態からのスタート

---

## プロダクト概要

### 何を提供するか

| 要素 | 内容 |
|------|------|
| 予想思考プロセスの記録 | レースごとに「なぜその予想にしたか」を構造化して記録 |
| 選手評価データベース | 独自の評価軸で全選手を体系的に評価・更新 |
| 評価方法の公開 | 選手をどう見るか、何を重視するかの方法論を公開 |
| AI予想との比較 | 人間の思考とAI予想を並べて違いを学べる |

### 誰に届けるか（ターゲット）

- **メインターゲット**: 競輪を始めて1〜3年、もっと上手くなりたい中級者
- **サブターゲット**: 競輪に興味があるが予想の仕方が分からない初心者
- **将来ターゲット**: 自分の予想を体系化したい上級者

## コア機能

### 1. 予想思考プロセスの記録・公開

レースごとに以下を構造化して記録する：

```
【レース概要】
- 開催場・グレード・レース番号

【ライン構成の読み】
- どのラインが形成されるか
- なぜそう読んだか（地区、師弟関係、過去の連携など）

【展開予想】
- 誰が主導権を取るか
- どのタイミングで仕掛けるか
- 想定される展開パターン（本線/対抗/穴）

【選手個別評価】
- このレースにおける各選手の状態・力量判断
- 注目ポイント（直近の調子、バンク相性、etc.）

【最終予想とその根拠】
- 印の配分と理由
- 買い目の組み立て方
```

### 2. 選手評価システム

既存の選手評価データモデルを活かし、以下を提供：

- **評価軸の定義と説明**: 各評価軸（スピード、仕掛け、スタミナ、競輪IQ等）が何を意味するか
- **評価の更新履歴**: いつ・なぜ評価を変更したかの記録
- **評価の根拠**: 具体的なレース映像やデータに基づく説明

### 3. 評価方法の教育コンテンツ

- **選手の見方講座**: 何を基準に選手を評価するか
- **展開の読み方講座**: ライン構成・展開予想の考え方
- **バンク特性の解説**: 各競輪場の特徴と予想への影響
- **買い目の組み立て方**: 予想を買い目に変換するプロセス

## サブスクリプションモデル

### プラン構成（案）

| プラン | 内容 | 想定価格帯 |
|--------|------|-----------|
| 無料プラン | 一部レースの予想閲覧、選手評価の一部閲覧 | ¥0 |
| スタンダード | 全レース予想閲覧、全選手評価、思考プロセス全公開 | 未定 |
| プレミアム | スタンダード + 教育コンテンツ、質問機能、リアルタイム予想 | 未定 |

### マネタイズの考え方

- 価値の源泉は **「予想の結果」ではなく「予想の過程と方法論」**
- ユーザーが自分で考えられるようになることがゴール
- 的中率を売りにしない（学びを売る）

## 技術的な実現方針

### 既存資産の活用

| 既存 | 活用方法 |
|------|----------|
| 選手評価CSV | 選手評価DBの初期データとして取り込み |
| 出走表JSON | レースデータの構造化基盤 |
| docs/静的HTML | 予想ボード・バンク表示をサブスク版UIのプロトタイプに |
| RAGシステム | AI予想生成の基盤 |
| スクレイパー | 出走表・レース結果の自動取得 |

### 追加で必要なもの

- **認証・課金基盤**: ユーザー管理、サブスク決済
- **コンテンツ管理**: 予想記事・教育コンテンツの投稿・管理
- **通知システム**: レース前の予想配信
- **閲覧制御**: プラン別のコンテンツアクセス制御
